## Хочу смотреть

Если нужный сериал уже есть в [библиотеке переводов](http://84.38.182.45:5001/#translations) —
просто установи расширение и смотри. Без API-ключей, без регистрации.

1. Скачай [extension.zip](http://84.38.182.45:5001/extension.zip) и распакуй
2. Открой `chrome://extensions/`, включи «Режим разработчика»
3. «Загрузить распакованное» → выбери папку
4. Открой видео — субтитры подтянутся автоматически

> Скоро в Chrome Web Store

## Хочу переводить

У тебя подписка Claude Max?
Переводи то, что смотришь сам — перевод попадёт в библиотеку.

### Вариант 1: Через CLI (бесплатно при наличии подписки)

Подробная инструкция: [docs/CONTRIBUTING.md](docs/CONTRIBUTING.md)

### Вариант 2: Через OpenRouter API (платно)

1. Получи ключ на [openrouter.ai](https://openrouter.ai/)
2. Вставь ключ в настройках расширения
3. Выбери модель и язык → перевод запустится автоматически

> Топовые модели (Opus) стоят $5–15 за часовой эпизод.
> Экономный вариант: Gemini Flash, Haiku — дешевле, но качество ниже.

## Возможности

- **Работает с HLS-потоками** — проверено на Кинопабе, архитектурно не привязан к конкретному сервису
- **Библиотека переводов** — результаты сохраняются и становятся доступны через расширение
- **Двойные субтитры** — оригинал + перевод одновременно
- **Перевод на любой язык** — русский по умолчанию, но можно выбрать любой из списка в пикере
- **Выбор AI-модели** — Claude Sonnet 4.6, Claude Opus 4.6, Gemini 2.5 Flash, Llama 4 через OpenRouter. Через CLI: Opus, Sonnet, Haiku
- **Drag & drop субтитров** — перетащи .srt/.vtt в side panel для перевода через очередь
- **Настройка вида** — шрифт, цвет, прозрачность, позиция
- **Подстройка тайминга** — `[` / `]` для сдвига ±0.5с
- **Экспорт** — скачивание перевода в .srt или .vtt

## Как это работает

<details>
<summary>Техническая архитектура</summary>

```
  Браузер                 Расширение
  (Кинопаб и др.)         (background.js + content.js)
      |                        |
      |  Плеер загружает       |
      |  .m3u8 субтитры        |
      |───────────────────────>|
      |                        |
      |                  background.js:
      |                  - ловит URL через webRequest
      |                  - скачивает .vtt сегменты (обход CORS)
      |                  - склеивает в один VTT
      |                  - разбивает на батчи по 200 строк
      |                  - переводит через OpenRouter API
      |                  - gzip-кеширует в chrome.storage
      |                        |
      |  content.js:           |
      |  - рисует субтитры     |
      |    поверх видео        |
      |  - обновляет по        |
      |    requestAnimationFrame
      |<───────────────────────|
```

1. **Детекция** — service worker ловит `.m3u8` запросы через `chrome.webRequest`
2. **Скачивание** — background скачивает HLS-плейлист и все VTT-сегменты (обходит CORS)
3. **Склейка** — сегменты объединяются в один VTT с перенумерацией
4. **Перевод** — батчи по 200 строк отправляются на OpenRouter API
5. **Кеш** — переведённый VTT сжимается gzip и сохраняется в `chrome.storage.local`
6. **Рендер** — content script показывает субтитры синхронно с `video.currentTime`

Подробнее: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)

</details>

## FAQ

### Нужен ли API-ключ?

Нет, если нужный перевод уже есть в библиотеке. Ключ нужен только для перевода нового контента.

### Это бесплатно?

Да. Расширение бесплатное, библиотека переводов бесплатная.
Платить нужно только за API если хотите переводить сами через OpenRouter.
Если у вас есть подписка Claude Max — переводите бесплатно через Claude Code.

### Какое качество перевода?

Зависит от модели. Claude Opus 4.6 и Sonnet 4.6 дают качество на уровне хорошего фансаба — с юмором, сленгом и контекстом. Экономные модели (Gemini Flash, Haiku) — проще, но тоже читаемо.

### С какими сайтами работает?

Проверено на Кинопабе. Архитектурно работает с любым сайтом, где субтитры отдаются как HLS-поток.

### Работает в фуллскрине?

Да. Picker, субтитры и бейджик адаптируются к полноэкранному режиму.
